这几年折腾博客的总结

发表于2024-10-14|更新于2024-10-14|随笔
|总字数:1.6k|阅读时长:4分钟|浏览量:

上次讨论博客今后的发展方向也是一年前了，在此有些感想想记下来，同时我也在思考，之后我的这些网站要何去何从。

目前的网站

这三个是已经运行了很久的网站了

  • 个人主页：https://zerolacqua.top/

    主要是用来展示一些个人信息，以及一些链接。是直接用的别人的主页模板，纯 html 写的。

  • 博客：https://blog.zerolacqua.top/

    是我最开始折腾的网页，先是在 github 上部署，后迁移到服务器上运行至今。使用了 hexo 进行静态网页生成，引擎是基于 node.js 的。

  • 笔记：https://note.zerolacqua.top/

    是后来为了方便记录一些学习笔记而开的网站，使用了 mkdocs （material 主题）进行静态网页生成。引擎是基于 python 的。

还有一个是最近才搭好的博客

  • 新博客：https://vanblog.zerolacqua.top/

    最近搭建新博客，基于 vanblog 的 docker 镜像搭建的动态博客。因为有后端所以管理起来更容易，写起博文也更方便。技术项目什么可能会放在这个博客上吧。

还有一些网站是没部署的，或者是依附具体项目的

  • cdn：https://cdn.zerolacqua.top/

    用来存放一些图片、文件等静态资源，主要是博客和笔记的图片。同时也包括一些 js 脚本和样式表。搭建的原因是免费的公共 cdn 经常会抽风，可怜的 cdn 乞丐只能四处寻找能用的 cdn，最后实在受不了了自己整了个自建的。不过期间也遭遇过盗刷的问题。

  • docs：https://zerolacqua.github.io/tiny-matrix/

    由 doxygen 生成的静态文档，用来记录一个项目的代码文档，是一个简单的矩阵库。

  • 学术主页：scholar.zerolacqua.top

    用来展示一些学术信息，包括论文、项目等。使用 Hugo 进行静态网页生成，引擎是基于 go 的。不过这个网站并没有部署，因为我没什么学术成果（苦笑），简历啥的也不打算往上面放了。

目前遇到的问题

一是网站太多，维护起来很麻烦。每个网站都有自己的源文件，使用的引擎也各不相同。并且已有的网站都是静态生成的，所以每次更新都要重新生成一遍，然后再上传到服务器上。这个过程很繁琐。

个人主页因为是纯静态而且也没怎么更新，所以问题不大。笔记网站部署依靠 github action 自动完成（也许构建慢但我看不到所以无所谓？），而且本身 material for mkdocs 的颜值真的很不错，作为一个放文档的网站深得我心，所以目前一直用来放读书、论文和课程笔记。

而基于 hexo 的 旧博客 就比较尴尬了，虽然它在本地也有自动化脚本，但是我现在三十多篇文章每次生成都需要很久，真的很花时间，也不知道是不是我插件太多了，总之很头疼。而且每次写一篇文章我都需要为配图什么事情折腾半天，选好图以后要上传到 cdn 上去。顺便提一嘴，我的图片都是还没压缩的，因为越来越懒了。听说 Hugo 的构建速度会快很多，但是 Hugo 当时我折腾的时候还没有好看的主题，用作学术博客的主题也是我找了好一会才找到的，所以我也不想换 Hugo

二是定位不太清晰。前面提到了笔记网站放读书、论文和课程笔记（而且那个网站上我不怎么放图片）。而 blog 目前乱七八糟的什么都有，工作流也是最麻烦的，导致我虽然有一些写文的 idea，但很难付诸实践。一年前还只有笔记网站和旧博客的时候，我还勉强能分得清定位。但是现在有了一个动态的新博客，笔记网站里应该放什么就已经有点需要掂量了，旧博客的处境就更尴尬了。旧博客里面一些偏项目的博文已经被迁移到新博客下了，似乎没什么新内容能写在旧博客里了。那？旧博客应该怎么办？我应该放什么东西？

理论上新博客自带图床，所有的文章都能放在新博客上，而且支持上传自定义多文件页面（可以很方便地放个人项目的 demo），也允许一些自定义样式。旧博客唯一的优势可能就是插件很多，有非常多的样式插件，能折腾的东西很多，自定义程度很高。vanblog 支持的文章内的样式还挺少，就是个一般的 markdown 在线编译器。但是我现在也不怎么折腾样式了，而且 vanblog 后续可能也会更新一些新东西（会的吧，应该吧）。其实之前就已经看同学在用 vanblog 写博文了，当时是因为旧博客已经有一些规模了，所以一直懒得动。但是现在实在是受不了了，自己也搭了个 vanblog，体验意外的还不错？

总结

随着网站越来越多，我逐渐开始变得迷茫，我搭建博客的最初目的是为了记录自己的学习和生活，但是现在维护越来越麻烦，我也越来越懒。以前这个旧博客是我尝试折腾新技术新东西的地方，但是我现在我对新奇的样式已没了当年的热情，这个博客也成了束缚我的“旧东西”。或许我再也不会像大三大四闲时间一大把的时候那样，一折腾就是几天了,但是毕竟这个旧博客三年以来承载了我太多的快乐与成就感，我还是舍不得。或许我还会在这里写些更有生活味的东西，将它作为一个生活记录的小站；或许我会发现兜兜转转回到原点，静态博客还是更香更有技术味儿；或许它会是一个备份，我青春的备份，会是个纪念馆，用来怀念我那已不再的本科时光。

备份，也该备个份了，有空备份一下我这些网站源文件吧……
